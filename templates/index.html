🏥 Minicompilador Médico

Este programa es una aplicación web con Flask que simula un compilador de tres fases. Recibe un archivo .txt con datos del paciente, lo analiza léxicamente (reconociendo tokens mediante AFDs), sintácticamente (presencia de todos los campos) y semánticamente (validaciones de rango, formato, etc.). A continuación se muestran todos los autómatas y el árbol sintáctico.

📝 Formato esperado del archivo .txt: 
Nombre: Juan
Apellido: Pérez
DNI: 12345678
Edad: 30
Diagnóstico: J00
Fecha: 2025-06-10
Hora: 09:15
Hospital/Clínica: Clínica Santa María
Laboratorio: Lab Central
Salón: MI-P2-103
Exámenes: Hemograma, Perfil lipídico
Enfermera/Médico: Dra. Patricia López
Tipo de sangre: A+
Todos los campos son obligatorios.

🔍 AUTÓMATAS FINITOS DETERMINISTAS (AFD) PARA CADA TOKEN

📌 AFD para CODIGO_MEDICO (formato: [A-Z][0-9]{2})

Estados: q0 (inicial) → q1 → q2 → q3 (aceptación). Error: q_error muerto.

    ┌─────┐ A-Z   ┌─────┐ 0-9   ┌─────┐ 0-9   ┌─────┐
    │  q0 │ ────→ │  q1 │ ────→ │  q2 │ ────→ │  q3 │ (aceptación)
    └─────┘       └─────┘       └─────┘       └─────┘
      │                                         │
      │ otro                                     │ otro
      ▼                                         ▼
   ┌─────┐                                   ┌─────┐
   │error│                                   │error│
   └─────┘                                   └─────┘

Graphviz code:

digraph AFD_CODIGO {
    rankdir=LR;
    q0 [label="q0"]; q1 [label="q1"]; q2 [label="q2"];
    q3 [label="q3", shape=doublecircle];
    error [label="q_error"];
    q0 -> q1 [label="[A-Z]"];
    q0 -> error [label="otro"];
    q1 -> q2 [label="[0-9]"];
    q1 -> error [label="otro"];
    q2 -> q3 [label="[0-9]"];
    q2 -> error [label="otro"];
    q3 -> error [label="cualquier símbolo"];
}
📌 AFD para FECHA (YYYY-MM-DD)

Estados: q0 (inicial) → q1 → q2 → q3 → q4 → (guión) q5 → q6 → q7 → (guión) q8 → q9 (aceptación).

q0 --[0-9]--> q1 --[0-9]--> q2 --[0-9]--> q3 --[0-9]--> q4 --'-'--> q5
q5 --[0-9]--> q6 --[0-9]--> q7 --'-'--> q8 --[0-9]--> q9 (aceptación)

Graphviz code:

digraph AFD_FECHA {
    rankdir=LR;
    q0,q1,q2,q3,q4,q5,q6,q7,q8 [shape=circle];
    q9 [shape=doublecircle];
    q0 -> q1 [label="[0-9]"]; q1 -> q2 [label="[0-9]"];
    q2 -> q3 [label="[0-9]"]; q3 -> q4 [label="[0-9]"];
    q4 -> q5 [label="'-'"];   q5 -> q6 [label="[0-9]"];
    q6 -> q7 [label="[0-9]"]; q7 -> q8 [label="'-'"];
    q8 -> q9 [label="[0-9]"];
}
📌 AFD para HORA (HH:MM, formato 24h)

Estados: q0 → según primer dígito (0-1 o 2) → q2 → ':' → q3 → [0-5] → q4 → [0-9] → aceptación.

q0 --[0-1]--> q1a --[0-9]--> q2
q0 --'2'--> q1b --[0-3]--> q2
q2 --':'--> q3 --[0-5]--> q4 --[0-9]--> q5 (aceptación)

Graphviz code:

digraph AFD_HORA {
    rankdir=LR;
    q0,q1a,q1b,q2,q3,q4 [shape=circle];
    q5 [shape=doublecircle];
    q0 -> q1a [label="[0-1]"]; q0 -> q1b [label="'2'"];
    q1a -> q2 [label="[0-9]"]; q1b -> q2 [label="[0-3]"];
    q2 -> q3 [label="':'"];    q3 -> q4 [label="[0-5]"];
    q4 -> q5 [label="[0-9]"];
}

3 Traducción (código Python)

▸ GENERACIÓN DE CÓDIGO

  • Se construye un diccionario con los datos validados.
  • Se aplican expresiones regulares para validar formato de campos.
  • Se busca el diagnóstico en el diccionario CIE-10 y se decodifica.
  • Salida: mensaje de éxito con todos los datos del paciente o lista de errores clasificados por fase.

✅ Validaciones semánticas adicionales:

  • Edad: entero entre 0 y 120
  • DNI: exactamente 8 dígitos numéricos
  • Diagnóstico: código debe existir en el diccionario CIE-10
  • Salón: formato ESPECIALIDAD-P[PISO]-[NUMERO] (MI, CIR, PED, GO; piso 1-10, número 1-999)
  • Fecha: válida en formato YYYY-MM-DD
  • Hora: válida en formato HH:MM (24h)
  • Tipo de sangre: A+, A-, B+, B-, AB+, AB-, O+, O-
